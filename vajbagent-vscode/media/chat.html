Kako povezati MCP servise
Otvori fajl .vajbagent/mcp.json u tvom projektu (klikni "Konfiguriši") i dodaj servise koje zelis. Ispod su primeri za svaki servis — kopiraj kod i zameni vrednosti oznacene sa TVOJ_...
📁 File System (pristup fajlovima)
Daje agentu pristup fajlovima na tvom racunaru. Zameni putanju sa folderom koji zelis.
"filesystem": { "command": "npx", "args": [ "-y", "@modelcontextprotocol/server-filesystem", "TVOJ_FOLDER_PUTANJA" ] }
Primer putanje: /Users/ime/Documents ili C:\\Users\\ime\\Documents
⚡ Supabase (baza podataka)
  1. Idi na supabase.com/dashboard/account/tokens
  2. Klikni Generate new token i kopiraj token
  3. Project Ref nadjes u URL-u dashboarda: supabase.com/dashboard/project/TVOJ_REF
"supabase": { "command": "npx", "args": [ "-y", "@supabase/mcp-server-supabase", "--project-ref", "TVOJ_PROJECT_REF" ], "env": { "SUPABASE_ACCESS_TOKEN": "TVOJ_ACCESS_TOKEN" } }
Dodaj "--read-only" u args ako zelis samo citanje (bez menjanja baze).
🐙 GitHub (repozitorijumi, issues, PR)
  1. Idi na github.com/settings/tokens
  2. Klikni Generate new token (classic)
  3. Ukljuci permisije: repo, read:org
"github": { "command": "npx", "args": [ "-y", "@modelcontextprotocol/server-github" ], "env": { "GITHUB_PERSONAL_ACCESS_TOKEN": "TVOJ_GITHUB_TOKEN" } }
🌐 Netlify (deploy, hosting)
  1. Idi na app.netlify.com → User Settings → Applications
  2. Kreiraj Personal Access Token
"netlify": { "command": "npx", "args": ["-y", "@netlify/mcp"], "env": { "NETLIFY_PERSONAL_ACCESS_TOKEN": "TVOJ_NETLIFY_TOKEN" } }
▲ Vercel (deploy, hosting)
  1. Idi na vercel.com/account/tokens
  2. Kreiraj novi token
"vercel": { "command": "npx", "args": ["-y", "vercel", "mcp", "start"], "env": { "VERCEL_TOKEN": "TVOJ_VERCEL_TOKEN" } }
Posle dodavanja, klikni Restartuj u podesavanjima.
